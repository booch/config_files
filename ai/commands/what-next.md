# What to Do Next Command

Review the current project status, recent changes, uncommitted/unfinished work, and TODO items.
Then suggest what we should work on next, prioritizing by importance and logical workflow progression.

## Session Title

If `/what-next` is invoked at the very start of a thread (its first request), name the session
after the work that actually gets undertaken — not "what" or a generic placeholder. Choose
the title once a direction is picked, and update it if the focus shifts, so the session is
easy to find later by what was worked on. Use whatever session-title mechanism the harness
provides.
